Bpm Object¶

class reamber.base.BpmObj.BpmObj(offset: float = 0.0, bpm: float = 120.0, metronome: int = 4)¶

Bases: reamber.base.TimedObj.TimedObj

A non-playable timed object that specifies the tempo of the map.

This is synonymous with Bpm Point, it’s named Object to make it consistent

static alignBpms(bpms: List[reamber.base.BpmObj.BpmObj], BEAT_ERROR_THRESHOLD: float = 0.001, BEAT_CORRECTION_FACTOR: float = 5.0) → List[reamber.base.BpmObj.BpmObj]¶

Ensures that all BPMs are on an integer measure by adding or amending

Parameters

  • bpms – The BPMs

  • BEAT_ERROR_THRESHOLD – If the fraction change in beat’s is more than this, we append a new BPM, else we create a new bpm. i.e. if there’s too many high BPM points, then increase this

  • BEAT_CORRECTION_FACTOR – The number of beats to search prior to the affected beat to amend. i.e. if there are too many incorrectly snapped notes DUE TO ADDED BPMs then decrease this

Returns

The new and old BPMs

Snaps the offsets to the exact snap

Returns in the exact same order.

Example:

BpmObj.snapExact([1, 100, 250, 385], bpms=[BpmObj(0, 150)], snapPrecision=16)
[400.0, 300.0, 100.0, 0.0]

Note Object¶

class reamber.base.NoteObj.NoteObj(offset: float = 0.0, column: int = 0)¶

Bases: reamber.base.TimedObj.TimedObj

A Note Object is a playable timed object

Do not get confused with Hit Object, which is just a single hit/tap.

The naming convention is done this way to make it clear on what is a note, hit and hold.

class reamber.base.RAConst.RAConst¶

Bases: object

This class holds all constants that can be used throughout the program

The class defines a helper classes to convert in units:

hrToMin(10) == 600
minToSec(10) == 600
secToMSec(10) == 10000
secToHr(60 * 60) == 1

This also defines division classes, used in PlayField:
