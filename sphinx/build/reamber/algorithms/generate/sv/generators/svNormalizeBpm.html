SV Normalize Bpm¶

Let’s say we have the following:

OFFSET BPM
0      200
100    50
300    100

If the game decides that it’s reference BPM is 100; that is, it shows normal scroll speed during 100 BPM Sections.

Then you would call the algorithm as such

1
2
3
 # Assume osu is the map itself
 seq = svNormalizeBpm(osu.bpms, 100)
 svs = seq.writeAsSv(OsuSvObj)

svs would be the output.

reamber.algorithms.generate.sv.generators.svNormalizeBpm.svNormalizeBpm(bpms: reamber.base.lists.BpmList.BpmList, normalizeTo: float) → reamber.algorithms.generate.sv.SvSequence.SvSequence¶

Generates Normalizing SVs for Bpm.

Useful if Bpms are unexpectedly changing scroll speeds. Generated Scroll Velocities can counter it.

Parameters

  • bpms – Any Type of BpmList

  • normalizeTo – The Bpm to normalize to
